💡 HOW EACH OPTIMIZATION WORKS

Gzip Compression:
  1. Backend compresses response body
  2. Sends: Content-Encoding: gzip header
  3. Client automatically decompresses
  4. Result: 40-60% smaller responses ✅

Cache Headers:
  1. Server sets Cache-Control headers
  2. OkHttp stores response to disk
  3. Next identical request: returns cached copy (0 bytes!) ✅
  4. Timeout: 1 hour (private), 24 hours (public)

RunRepository:
  1. First call to getRunsForUser() → API call + cache
  2. Second call within 5 min → returns cache (0 API calls!) ✅
  3. Prevents duplicates across multiple ViewModels
  4. Example: 3 ViewModels requesting same runs
     - Before: 3 API calls
     - After: 1 API call + 2 cache hits (2/3 eliminated)
